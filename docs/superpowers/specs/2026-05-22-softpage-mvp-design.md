# SoftPage MVP 设计

## 概要
做一个固定比例的写作与导出工具，面向社媒图文内容。用户写一篇完整稿子，手动把图片插入到正文流里，预览会按 `3:4` 画布展示，超出后自动分页，最后按页导出多张 PNG。

## 目标
- 米白色背景：`#F6F1E8`
- 支持正文输入
- 支持基础排版参数：行高、字号、字重、段间距、页边距
- 支持手动上传图片并插入到指定段落后面
- 支持自动分页到多个 `3:4` 画布
- 支持按页导出多张 PNG

## 非目标
- 不做 Markdown 富渲染
- 不做拖拽式自由排版
- 不做多模板系统
- 不做账号、历史稿、云端存储

## 技术栈
- `Next.js`
- `html2canvas` 用于导出

## 页面结构
页面分三块：
- 顶部工具栏：导入图片、导出 PNG 和核心操作
- 左侧编辑区：正文编辑和排版参数
- 右侧预览区：固定 `3:4` 画布预览

## 内容模型
文章按线性块列表表示：
- `text` 块：一段正文
- `image` 块：上传图片，插入到指定段落后面

这样可以保持 MVP 简单，同时符合“正文流 + 图片插入”的写作方式。

## 编辑流程
1. 用户输入或粘贴完整稿子。
2. 用户上传图片。
3. 用户把图片插入到某个文本块后面。
4. 用户调整排版参数。
5. 预览实时更新。
6. 导出时按页生成 PNG。

## 分页规则
- 渲染器按块顺序排版。
- 每页使用固定 `3:4` 画布。
- 渲染前先测量块高度。
- 如果下一个块会溢出当前页，就移动到下一页。
- 分页在导出前完成，避免截图时内容被裁掉。

## 导出规则
- 导出是按页导出，不是整篇一次截图。
- 每一页单独用 `html2canvas` 截取。
- 结果是一组 PNG 文件，每页一张。
- 导出时要保留画布尺寸和背景色。

## 排版控制
支持以下控制项：
- 字号
- 行高
- 字重
- 段间距
- 页边距

这些控制项同时作用于预览和导出的 PNG。

## 错误处理
- 图片上传失败时，要给出明确报错，并保留当前文章内容。
- 如果分页测量出现偏差导致溢出，优先换页，不要裁切内容。
- 如果某一页导出失败，要提示失败页并停止批量导出。

## 验收标准
- 用户可以输入文本，并在 `3:4` 画布中看到结果。
- 用户可以把上传的图片插入到正文流里。
- 用户可以调整排版参数，并看到预览同步变化。
- 超过一页的内容会自动分页。
- 当文章超过一页时，导出会生成多张 PNG。
