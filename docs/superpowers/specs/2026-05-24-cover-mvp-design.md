# SoftPage 封面 MVP 设计

## 概要
在现有正文排版之外，补一套独立的封面制作功能。封面只服务 `9:16` 长文阅读模式，单篇文章只生成 `1` 张封面。封面支持从 Markdown 自动带出文章标题，支持手动上传一张主视觉图，支持单独导出封面 PNG。

## 目标
- 提供独立的封面制作工作区
- 支持从 Markdown 自动带出文章标题
- 标题支持手动修改
- 作者信息固定默认值为 `花萍雨`
- 支持手动上传 `1` 张封面主图
- 支持实时预览单张 `9:16` 白底封面
- 支持单独导出 `1` 张封面 PNG

## 非目标
- 不做多封面候选
- 不做封面模板切换
- 不做副标题、标签、栏目等额外文案位
- 不做从正文图片中选主图
- 不做封面和正文一键合并导出
- 不做图片滤镜、遮罩、渐变压字
- 不做复杂字体、字重、对齐控制面板
- 不做封面历史版本管理

## 页面结构
在现有编辑器中增加模式切换：
- `正文排版`
- `封面制作`

默认进入 `正文排版`。切换到 `封面制作` 后：
- 左侧显示封面编辑面板
- 右侧显示封面实时预览
- 正文分页预览暂时隐藏

## 输入项
封面数据独立于正文 `blocks`，只包含以下字段：
- `title`：默认从 Markdown 标题带出，可手动修改
- `author`：默认值 `花萍雨`
- `heroImage`：手动上传的封面主图
- `hasDivider`：是否显示底部分割线，默认 `true`

第一版不开放以下输入：
- 图片裁切焦点调节
- 标题字号、字重、对齐切换
- 作者名编辑开关

## 版式规则
封面固定为 `9:16` 白底画布，并拆成五个纵向区域：

| 区域 | 大致高度 | 占比 |
|---|---:|---:|
| 顶部留白 | 150 px | 8% |
| 主视觉图 | 820 px | 46% |
| 标题区 | 420 px | 23% |
| 作者信息区 | 120 px | 7% |
| 底部留白 + 分割线 | 290 px | 16% |

对应规则：
- 顶部留白不放内容，只负责呼吸感
- 主视觉图位于顶部留白之后，宽度铺满版心，默认 `cover` 裁切，使用轻圆角
- 标题区放在主图下方，左对齐，支持自动换行，也允许手动断行
- 标题建议控制在 `2-4` 行内，超过后仍显示，但不为第一版单独做超长标题优化
- 作者信息区单独占位，默认文案固定为 `| 作者：花萍雨`
- 底部留白保留足够空间，分割线位于底部区域靠下位置

## 编辑流程
1. 用户导入 Markdown 原稿。
2. 系统自动提取文章标题，写入封面标题默认值。
3. 用户切换到 `封面制作`。
4. 用户上传封面主图。
5. 用户按需要修改标题断行或标题文案。
6. 用户单独导出封面 PNG。

## 导出规则
- 封面导出与正文导出分离
- 只导出当前封面预览，不参与正文 ZIP
- 导出产物是 `1` 张 PNG
- 导出尺寸与预览一致，背景为纯白

## 与正文系统的关系
- 正文排版继续走现有 `Markdown -> blocks -> paginate -> pages` 链路
- 封面走独立链路：`Markdown 标题 -> cover draft -> cover preview -> cover export`
- 两条链路共享 Markdown 标题来源，但不共享图片和分页逻辑
- 修改封面不会影响正文分页
- 修改正文也不会直接覆盖用户手动改过的封面标题，除非用户重新导入 Markdown 并接受新的默认标题

## 错误处理
- 未上传主图时，预览区显示占位状态，但仍允许先编辑标题
- 主图上传失败时给出明确报错，不清空现有标题内容
- 如果 Markdown 中未识别出标题，则封面标题留空，由用户手动填写
- 封面导出失败时单独提示，不影响正文导出功能

## 验收标准
- 用户可以在现有编辑器中切换到 `封面制作`
- Markdown 标题可以自动带入封面标题输入框
- 用户可以手动上传 `1` 张主图并看到实时预览
- 封面预览遵守五区版式规则
- 用户可以单独导出 `1` 张封面 PNG
